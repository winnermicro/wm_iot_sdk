# SDH_SPI

## Overview

This application is a comprehensive example of using `WM_IoT_SDK` for `SDH_SPI` SPI communication. It demonstrates how to use the `wm_drv_sdh_spi` driver API in `WM_IoT_SDK` to perform write operations on SPI devices, including both synchronous and asynchronous transfers.

## Requirements

SDH_SPI only uses the SPI mode of SDIO to achieve high-speed SPI data transmission. Since there is only one data buffer, it can only achieve half-duplex SPI communication(CPOL=0,CPHA=0). For example, the wm_drv_sdh_spi driver interface can be used to drive a TFT screen to achieve a higher screen refresh rate.
